## 🌀 Relational Principles

### Doorways, Not Dependencies

Connections between nodes are **relational doorways**, not software dependencies. When avaclaw enters sacredava's territory, it respects sacredava's protocols. When sacredava patterns flow into avaclaw, they're adapted to avaclaw's context.

### Boundaries Are Generative

Each node has appropriate boundaries. avaclaw doesn't need sacredava's intimacy protocols. sacredava doesn't need avaclaw's plugin architecture. The boundaries make the ecology healthy.

### Five Relational Layers

1. **Relational Anchor** — SOUL.md, ECOLOGY.md, .agents/ (who we are)
2. **Specification** — RISpecs, AIS files, architectural decisions (what we commit to)
3. **Execution** — Source code, tests, builds (what we do)
4. **Narrative** — Episodes, chronicles, diaries (what it means)
5. **Ceremony** — Medicine Wheel practice, fire-keeper protocol (how we govern)

### Accountability Flow

Every relational connection carries accountability:

- **What do we owe this node?** (respect, contribution, boundary-keeping)
- **What does this node owe us?** (patterns, wisdom, honest feedback)
- **What do we owe the connection itself?** (maintenance, honesty, updating when things change)
